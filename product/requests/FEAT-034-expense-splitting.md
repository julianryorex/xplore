## Problem

Group trips involve shared costs; users already use Splitwise in parallel. Adjacent job-to-be-done increases stickiness.

## Proposed solution

Simple trip ledger: add expense, split equally or by member, running balances. Export summary at trip end. Deep link to Splitwise optional.
